# مولّد الصور الوصفية للمتاجر — AppsBunches

أداة ويب (تعمل بالكامل في المتصفح، بدون سيرفر) تأخذ سكرينشوتات تطبيقك وتنتج صورًا تسويقية
جاهزة للرفع على **App Store** و **Google Play** بالأحجام المعتمدة، مع توليد الأيقونات وصورة الكفر.

## المميزات

- **قوالب جاهزة** بستايل احترافي: خلفية ثيم + عنوان عربي + إطار جوال + السكرينشوت.
- **شريط حالة موحّد** يُرسم فوق الشريط الحقيقي، بنمطين مختلفين: **iOS** و **Android**،
  مع تحديد لون الأيقونات (أسود/أبيض) **تلقائيًا** حسب سطوع الخلفية.
- **الأحجام المعتمدة**: App Store 6.5″ و 6.7″، Google Play 1080×1920، الكفر 1024×500،
  والأيقونات 1024 / 512 / 540.
- **تصدير الكل دفعة واحدة** في ملف `store-assets.zip`.
- واجهة عربية RTL بثيم AppsBunches.

## التشغيل محليًا

تحتاج خادمًا محليًا (لأن وحدات JS والصور تتطلب `http://` وليس `file://`):

```bash
cd store-asset-generator
python3 -m http.server 8000
# ثم افتح: http://localhost:8000
```

## الاستخدام

1. ارفع سكرينشوت أو أكثر (نفس المنصّة).
2. اختر **منصّة شريط الحالة** (آيفون/أندرويد).
3. اكتب العنوان واختر **خلفية الثيم**.
4. (اختياري) ارفع **اللوجو** لتوليد الأيقونات والكفر.
5. حدّد **الأحجام المطلوبة** ثم اضغط **تصدير الكل (ZIP)**.

## النشر على GitHub Pages

الأداة Vanilla JS بدون خطوة build:

1. ادفع المشروع إلى مستودع GitHub.
2. Settings → Pages → Source = `main` / root.
3. الرابط: `https://<username>.github.io/<repo>/`.

## البنية

```
index.html        # الواجهة (RTL)
css/styles.css    # ثيم AppsBunches
js/
  app.js          # إدارة الحالة وربط الأحداث
  presets.js      # الأحجام المعتمدة
  themes.js       # ثيمات خلفيات الصور
  renderer.js     # محرّك الرسم + أدوات مساعدة
  deviceFrame.js  # إطار الجوال (vector)
  statusBar.js    # شريط الحالة iOS / Android
  templates/      # قوالب: screenshot / icon / feature
  export.js       # تجميع الصور في ZIP
lib/jszip.min.js  # مكتبة ZIP (vendored)
assets/logo/      # شعار AppsBunches
```
